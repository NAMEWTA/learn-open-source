# 开源项目深度学习

系统性引导用户从全景到细节地学习一个开源项目，产出结构化笔记，最终具备二次开发能力。

## 核心原则

- 所有交互和产出使用中文
- 子模块只读，绝不修改子模块内任何文件
- 所有笔记输出到 `notes/<项目名>/` 目录
- 引用代码时复制代码块并标注完整文件路径
- 不确定的内容坦诚说明，不编造不猜测
- 大量使用 Mermaid 和 ASCII 图表辅助理解（参见 [references/visualization-guide.md](references/visualization-guide.md)）

## 工作流程

### 阶段一：项目探索与目录生成

1. 读取项目根目录，分析目录结构、README、配置文件、依赖声明
2. 识别技术栈、项目定位、核心模块
3. 生成学习目录（作为 `notes/<项目名>/README.md`），格式：

```markdown
# <项目名> 学习笔记

> 来源：<仓库地址>
> 简介：<一句话描述>
> 技术栈：<主要语言和框架>

## 学习目录

- [ ] 01-项目全景.md — 定位、技术栈、目录结构、依赖关系
- [ ] 02-架构拆解.md — 模块划分、数据流、核心抽象
- [ ] 03-核心模块深入.md — 关键实现、设计模式、算法
- [ ] 04-运行机制.md — 启动流程、请求链路、状态管理
- [ ] 05-二次开发指南.md — 扩展点、插件机制、实战演练
```

4. 根据项目复杂度自行决定是否拆分更多章节（如将核心模块拆为多章）
5. 将目录展示给用户确认，等待用户同意或调整后再继续

### 阶段二：逐章生成

确认目录后，按顺序逐章生成笔记。每章完成后：

1. 将笔记写入 `notes/<项目名>/XX-主题名.md`
2. 更新 `notes/<项目名>/README.md`，将对应条目打勾：`- [x]`
3. 告知用户本章已完成，询问是否有疑问或需要补充
4. 用户可提问、要求深入某个细节、或补充内容到当前章节
5. 用户确认无问题后，继续下一章

### 阶段三：收尾

所有章节完成后：
1. 确认 README.md 中所有条目已打勾
2. 更新 `notes/README.md` 总进度（如果存在）

## 章节生成规范

每章笔记必须包含以下结构（详细模板参见 [references/chapter-templates.md](references/chapter-templates.md)）：

```markdown
# 主题名称

## 学习目标
本章要搞清楚什么（2-3 句话）

## 正文内容
（根据层级不同，内容侧重不同，见下方五层说明）

## 检查点
1. 问题 1（验证理解的关键问题）
2. 问题 2
3. 问题 3
```

### 正文要求

- 代码引用：复制关键代码片段，标注完整路径
  ```
  > 文件：`deer-flow/src/agents/base.py`
  ```
- 图表：每章至少包含 1 个 Mermaid 或 ASCII 图表
  - 架构关系 → Mermaid graph
  - 请求链路 → Mermaid sequenceDiagram
  - 启动/业务流程 → Mermaid flowchart
  - 类继承 → Mermaid classDiagram
  - 状态流转 → Mermaid stateDiagram
  - 目录结构 → ASCII 树
  - 数据管道 → ASCII 流程图
- 检查点：每章结尾 3-5 个问题，帮助用户验证是否真正理解

### 五层学习路线

根据项目实际情况灵活调整，不必严格对应五个文件：

| 层级 | 主题 | 核心内容 |
|------|------|---------|
| 第一层 | 项目全景 | 定位、技术栈、目录结构、核心依赖、快速上手 |
| 第二层 | 架构拆解 | 模块划分、数据流、核心抽象、配置体系 |
| 第三层 | 核心模块 | 关键实现、设计模式、调用链路、算法分析 |
| 第四层 | 运行机制 | 启动流程、请求生命周期、状态管理、错误处理 |
| 第五层 | 二次开发 | 扩展点、插件机制、实战演练、开发环境 |

## 交互规范

- 目录确认前不生成任何章节内容
- 每章生成后暂停，等待用户反馈
- 用户提问时基于源码如实回答，不确定就说不确定
- 用户要求补充时，直接追加到当前章节文件中
- 目录可随时调整：用户可要求新增、合并、拆分、重排章节
- 调整目录后同步更新 README.md

## 文件命名

- 索引：`notes/<项目名>/README.md`
- 笔记：`notes/<项目名>/XX-主题名.md`（XX 为两位数序号）
- 文件名使用英文或拼音，内容使用中文
