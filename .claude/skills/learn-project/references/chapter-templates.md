# 章节模板参考

各学习层级的详细模板，供生成笔记时参考。所有笔记输出到 `notes/<项目名>/` 目录。

## 目录索引模板（README.md）

```markdown
# <项目名> 学习笔记

> 来源：<仓库地址>
> 简介：<一句话描述>
> 技术栈：<主要语言和框架>

## 学习目录

- [ ] 01-项目全景.md — 定位、技术栈、目录结构、依赖关系
- [ ] 02-架构拆解.md — 模块划分、数据流、核心抽象
- [ ] 03-核心模块深入.md — 关键实现、设计模式、算法
- [ ] 04-运行机制.md — 启动流程、请求链路、状态管理
- [ ] 05-二次开发指南.md — 扩展点、插件机制、实战演练
- [ ] ...（根据项目复杂度可拆分更多章节）

## 学习状态

- 开始时间：YYYY-MM-DD
- 当前进度：第 X 章
```

## 第一层：项目全景（01-项目全景.md）

```markdown
# 项目全景

## 学习目标
搞清楚这个项目是什么、解决什么问题、用了哪些技术

## 项目定位
- 一句话描述项目用途
- 目标用户/使用场景
- 与同类项目的差异

## 技术栈总览
| 层级 | 技术 | 用途 |
|------|------|------|
| 前端 | ... | ... |
| 后端 | ... | ... |
| 数据 | ... | ... |
| 部署 | ... | ... |

## 目录结构
（ASCII 目录树，标注每个顶层目录的职责）

## 核心依赖分析
列出 package.json / requirements.txt / go.mod 中的关键依赖及其作用

## 快速上手
项目的安装、配置、运行步骤（引用官方文档）

## 检查点
1. 能否用一句话向别人解释这个项目做什么？
2. 能否画出项目的顶层目录结构并说明每个目录的职责？
3. 能否列出 3 个核心依赖并解释为什么需要它们？
```

> 文件：`<项目名>/path/to/base_class.py`
